# Разработка Backend: стек и оптимизация сервера

## Обзор

Backend проекта строится на **Java** с использованием экосистемы **Spring**. Архитектура ориентирована на надёжность, масштабируемость и удобство поддержки.

---

## Технологический стек

### Ядро приложения
- **Java 21** — современная LTS-версия платформы с актуальными возможностями языка.
- **Spring Boot 3.4** — каркас приложения: автоконфигурация, встроенный сервер, управление зависимостями и жизненным циклом компонентов.

### Веб и API
- **Spring Web** — REST API, маршрутизация, обработка HTTP-запросов.
- **Spring WebSocket** — двусторонняя связь в реальном времени (стриминг, уведомления, интерактивные сценарии).
- **SpringDoc OpenAPI** — описание и документация API в формате OpenAPI 3.0 с интерактивным UI (Swagger).

### Работа с данными
- **Spring Data JPA** — объектно-реляционное отображение и репозитории для доступа к БД.
- **PostgreSQL** — основная реляционная СУБД для хранения метаданных, связей и бизнес-сущностей.
- **Spring JDBC** — прямой доступ к БД там, где нужен контроль над SQL и производительностью.

### Хранение файлов и медиа
- **MinIO** — S3-совместимое объектное хранилище для файлов, изображений и медиа-контента.

### Безопасность и валидация
- **JWT (JSON Web Tokens)** — аутентификация и авторизация без состояния (stateless).
- **Spring Validation** — валидация входящих данных на уровне DTO и контроллеров.

### Вспомогательные инструменты
- **MapStruct** — генерация мапперов между сущностями и DTO для чистого и производительного кода.
- **Lombok** — сокращение шаблонного кода (геттеры, сеттеры, билдеры, логирование).

---

## Архитектурные слои

- **Controller** — приём запросов, маршрутизация, преобразование в/из DTO.
- **Service** — бизнес-логика, оркестрация вызовов, транзакции.
- **Repository** — доступ к данным через JPA-репозитории и кастомные запросы.
- **Model / Entity** — доменные сущности и их отображение в БД.
- **DTO** — объекты для обмена с клиентом и между слоями.
- **Config** — конфигурация бинов (MinIO, WebSocket, CORS и т.д.).

---

## Инфраструктура и деплой

- **Docker / Docker Compose** — контейнеризация приложения, PostgreSQL, MinIO и сопутствующих сервисов.
- **Health checks** — проверка готовности БД и зависимых сервисов при старте.
- Управление ресурсами (в т.ч. GPU при необходимости) через конфигурацию деплоя.

---

## Оптимизация сервера

- Выбор подходящих индексов и запросов к PostgreSQL для снижения нагрузки и времени отклика.
- Использование пулов соединений к БД и к объектному хранилищу.
- Асинхронная обработка тяжёлых операций и стриминг через WebSocket там, где это уместно.
- Контейнеризация и изоляция сервисов для предсказуемой работы под нагрузкой.
